# 数据说明文档

本篇文档旨在说明本项目所使用的数据集来源、原始数据格式、整体数据规模、处理流程以及许可协议，并提供 3 种微调方案中的样本格式。

---

## 一、 数据来源、原始形式与许可/引用

### 1. 数据来源
本项目采用魔塔社区（ModelScope）开源的 **NLPCC 2017 单文档新闻摘要数据集**。该数据源自今日头条平台，涵盖了多个领域的真实新闻和人工撰写的摘要。

### 2. 原始数据集形式
原始数据集存储为 JSON 格式，其核心结构为包含在 `"data"` 数组中的“摘要-正文”键值对，其中 `"title"` 字段作为参考摘要，`"content"` 字段作为新闻正文。样例如下：

```json
{
  "data": [
    {
      "title": "知情人透露章子怡怀孕后,父母很高兴。章母已开始悉心照料。据悉,预产期大概是12月底",
      "content": "四海网讯,近日,有媒体报道称:章子怡真怀孕了!报道还援引知情人士消息称,“章子怡怀孕大概四五个月,预产期是年底前后,现在已经不接工作了。”这到底是怎么回事?消息是真是假?针对此消息,23日晚8时30分,华西都市报记者迅速联系上了与章子怡家里关系极好的知情人士,这位人士向华西都市报记者证实说:“子怡这次确实怀孕了。她已经36岁了,也该怀孕了。章子怡怀上汪峰的孩子后,子怡的父母亲十分高兴。子怡的母亲,已开始悉心照料女儿了。子怡的预产期大概是今年12月底。”当晚9时,华西都市报记者为了求证章子怡怀孕消息,又电话联系章子怡的亲哥哥章子男,但电话通了,一直没有人接听。有关章子怡怀孕的新闻自从2013年9月份章子怡和汪峰恋情以来,就被传N遍了!不过,时间跨入2015年,事情却发生着微妙的变化。2015年3月21日,章子怡担任制片人的电影《从天儿降》开机,在开机发布会上几张合影,让网友又燃起了好奇心:“章子怡真的怀孕了吗?”但后据证实,章子怡的“大肚照”只是影片宣传的噱头。过了四个月的7月22日,《太平轮》新一轮宣传,章子怡又被发现状态不佳,不时深呼吸,不自觉想捂住肚子,又觉得不妥。然后在8月的一天,章子怡和朋友吃饭,在酒店门口被风行工作室拍到了,疑似有孕在身!今年7月11日,汪峰本来在上海要举行演唱会,后来因为台风“灿鸿”取消了。而消息人士称,汪峰原来打算在演唱会上当着章子怡的面宣布重大消息,而且章子怡已经赴上海准备参加演唱会了，怎知遇到台风，只好延期，相信9月26日的演唱会应该还会有惊喜大白天下吧。"

```

### 3. 许可与引用说明
*   **许可协议**：该数据集由自然语言处理与中文计算会议（NLPCC）官方评测任务发布。仅供学术研究和教学目的使用，禁止用于任何商业用途。
*   **引用方式**：在研究或学术报告中使用此数据集时，需遵守 NLPCC 官方规范，引用 NLPCC 2017 Shared Task 1 (Single Document Summarization) 的相关评测标准。

## 二、 数据规模与划分

*   **原始数据规模**：包含约 **49,944** 组基础数据对。
*   **精筛后数据规模**：经过双重条件过滤和深度清洗后，最终保留 **5,695** 条高质量样本。
*   **数据集划分比例**：按照 **8:1:1** 的比例进行随机划分。
    *   **训练与验证集**：共 **5,126** 条样本（其中训练集 4,557 条，验证集 569 条）。
    *   **测试集**：共 **569** 条样本。
*   特别地，在双任务微调中，我们通过在训练集中引入 1:1 的辅助任务，其训练样本总量扩充至 **10,252** 条（包含 5,126 条摘要生成任务和 5,126 条核心原句提取任务）。

## 三、 数据处理、清洗与大模型生成流程

为了保证模型训练的稳定性和生成摘要的合规性，我们对原始数据进行了三步处理流程：

```text
 原始数据 (49,944条)
     │
     ▼
 1. 基础数据清洗 ──► 去除网页噪声、截断文末推广广告与非必要链接
     │
     ▼
 2. 规则条件筛选 ──► 过滤字数限制（正文1000-2000字，参考摘要20-60字）
     │
     ▼
 3. 大模型辅助构造 ──► 使用 deepseek v4-flash 生成思维链（CoT）、DPO偏好对及辅助任务目标
     │
     ▼
 最终数据集 (5,695条)
```

1.  **基础数据清洗**：
    *   对新闻正文部分进行扫描，定位并截断删除文末的推广链接及广告文案。
    *   过滤无意义字符、网页乱码及多余的连续空格。
2.  **规则条件筛选**：
    *   选择正文（`content`）字数在 $1000 \sim 2000$ 字之间的长篇新闻。
    *   选择原标题/参考摘要（`title`）字数在 $20 \sim 60$ 字之内的样本，确保参考摘要符合高度凝炼的硬性约束。
3.  **大模型辅助构造**：
    *   对于高级微调方案（思维链+DPO 微调、双任务微调），本系统**并未采用高成本的人工撰写方式**。
    *   我们利用 **deepseek v4-flash** 模型进行批量化、自动化的引导生成：
        *   **思维链构造**：调用 `deepseek v4-flash` 模型分析原文，提炼出逻辑推理过程（思维链）和核心关键词。
        *   **DPO 偏好对构建**：利用 `deepseek v4-flash` 生成推荐正样本（严谨推理与高度忠实摘要）与拒绝负样本（包含幻觉与胡乱捏造推理的摘要），用于偏好对齐训练。
        *   **双任务辅助目标**：调用 `deepseek v4-flash` 按照严格规则（如原封不动、不超过三句）从新闻正文中提取出代表核心内容的骨干原句。

## 五、 数据合规性声明

1.  **隐私脱敏**：该数据集在清洗过程中，对可能包含的个人非公开隐私、特定联系电话等敏感信息进行了去标识化处理。
2.  **大模型内容安全过滤**：经由 `deepseek v4-flash` 批量生成的数据均经过本地安全过滤和质量校验，确保生成的逻辑、语料安全可靠，不产生偏见、侵权或违规言论。
